HTML Basics

Getting Started

What is HTML?

HTML, which stands for HyperText Markup Language, is the standard markup language used to create web pages. It is the code that is used to structure a web page and its content. For example, content could be structured within a set of paragraphs, a list of bulleted points, or using images and data tables.

What is HTML for?

HTML is just like any tool that we use everyday. For example, drills are used to create holes while HTML are used to create websites. It defines and structures the content of your webpage, it’s that simple.

History

HTML was first developed by Tim Berners-Lee in 1991 to facilitate information sharing among scientists at CERN. HTML 2.0 was published in 1995, adding more capabilities to the original version. In 1997, HTML 3.2 and 4.0 were released by W3C, introducing CSS for styling. HTML 4.01, released in 1999, became widely adopted with minor updates. In the 2000s, focus shifted to XHTML for stricter coding practices. HTML5 development began in 2008 to improve multimedia support, a nd it became the official standard in 2014. HTML 5.1 and 5.2 were released in 2016 and 2017, respectively, with further enhancements.

Fundamentals of HTML

What is the HTML Syntax

Syntax refers to the set of rules that define the structure and composition of statements in a programming language. In the context of HTML, syntax dictates how HTML elements should be written and arranged to ensure that web browsers can correctly interpret and display the content.

Think of HTML syntax like the grammar rules of a language. Just as sentences need proper punctuation and structure to be understood, HTML elements need to follow specific rules to be correctly interpreted by web browsers.

So how does a proper HTML Syntax would look like?

The main part are as follows:

  1. The opening tag: This consists of the name of the element (in this case, p), wrapped in opening and closing angle brackets. This states where the element begins or starts to take effect — in this case where the paragraph begins.
  2. The closing tag: This is the same as the opening tag, except that it includes a forward slash before the element name. This states where the element ends — in this case where the paragraph ends. Failing to add a closing tag is one of the standard beginner errors and can lead to strange results.
  3. The content: This is the content of the element, which in this case, is just text.
  4. The Element tag: The opening tag, the closing tag, and the content together comprise the element.

What are tags?

Tags are what defines the elements that structure and present content on a web page. Each tag specifies a part of the document and instructs the browser on how to display it.

Think of HTML tags as labels in a folder of a filing cabinet. Just as labels categorize and organize documents in a filing cabinet, HTML tags organize and structure content on a web page.

What are elements?

HTML Elements are the individual components of an HTML document that define its structure and content. Each element consists of a start tag, content, and an end tag (except for self-closing/void elements).

As you may recall, we compared tags as labels in a folder of a filing cabinet Now, think of HTML elements as each folders inside a filing cabinet. Just as each folder organizes related documents, each HTML element organizes a specific piece of content on a web page.

HTML provides the structure of a webpage by using various elements, which are represented by tags. These tags tell the browser how to display the content on the page. HTML elements can include text, images, links, forms, and other types of content.

What are attributes?

HTML attributes provide additional information about HTML elements. They are always included in the opening tag of an element and usually come in name-value pairs, separated by an equals sign (=).

Think of HTML attributes like the extra notes or stickers you might put on a folder in an office. While the labels (tags) indicates what kind of documents a folder (elements) holds, the notes or stickers (attributes) provide extra details, such as the folder's importance, date, or any special instructions.

What is Nesting?

Nesting in HTML refers to placing one HTML element inside another. This hierarchical structure allows for the creation of complex layouts and ensures that elements are properly organized within the document.

Think of nesting like a filing system in an office. Just as folders (child elements) are placed within drawers (parent elements) of a filing cabinet to keep documents organized, HTML elements are nested within other elements to create a well-structured web page.

Some Common Tags in HTML

Paragraph Tags

The <p> tag in HTML is used to define a paragraph. It represents a block of text and is one of the most fundamental elements for structuring and formatting content in HTML documents.

Heading tags

Heading tags in HTML are used to define headings on a webpage. They range from <h1> to <h6>, with <h1> being the highest level (or most important) heading, and <h6> being the lowest level (or least important) heading. These tags help structure the content by breaking it into sections and providing a hierarchy of importance.

Think of heading tags like the titles and subtitles in a book. The main title of the book is like the <h1> tag, chapter titles are like <h2>, section titles within chapters are like <h3>, and so on. This hierarchy helps readers understand the structure and importance of different parts of the content.

Image Tags

Image tags in HTML are used to embed images in a web page. The <img> tag is an empty (self-closing) tag, meaning it doesn't have a closing tag. Instead, it uses attributes to specify the image source and provide additional information.

Think of an image tag like a picture frame. The frame (the <img> tag) holds the picture (the image file) and can have a label or description (attributes) that gives more context about the picture.

Anchor tags

Anchor tags in HTML, represented by the <a> tag, are used to create hyperlinks, which are clickable links that navigate to other web pages, files, or locations within the same page. The anchor tag is one of the most fundamental elements in HTML for linking content and enabling navigation.

Think of anchor tags like street signs or bookmarks. Just as a street sign points you to a different location or a bookmark takes you to a specific page in a book, an anchor tag directs users to another page or a specific part of the same page.

Division tags

The <div> tag in HTML is a block-level element used to group and organize content. It acts as a container for other HTML elements, allowing you to structure your webpage and apply styles or scripts to sections of your content.

Think of <div> tags like compartments in a drawer organizer. Just as the compartments help separate and organize different items within the drawer, <div> tags help organize and group different elements within a webpage.
